3. Tools to be Used :
Frontend:
• Vanilla JavaScript (ES Modules)
• Custom CSS Variables Layout
• HTML5 Canvas & SVG Visualizers
Backend & Database:
• Node.js & Express (TypeScript)
• Oracle Database 21c Express (XE)
• Docker & Docker Compose
4. Key Features :
  • i. Reversible Playback: Time-travel debugger executing step-snapshots forward and backward.
  • ii. Multi-Visualizer Layouts: 6 custom canvas/SVG layouts displaying bars, grids, matrices, strings, and graph shapes.
  • iii. Clean Architecture API: Loose coupling of Domain, Application use-cases, and Oracle Infrastructure repository queries.
  • iv. Explanations & Snippets: Dynamic DB-loaded 5-section descriptions and tabbed code widgets per algorithm.
2. Objective :

To design and implement a zero-dependency frontend Single Page Application (SPA) linked to a robust Clean Architecture REST API for algorithm visualization.

The project aims to separate pedagogical data ("what to teach") in a normalized database from visual state-generators ("how to animate") on the client, giving users interactive controls, progress updates, and time-travel tracing.

5. Target Audience :
  • Computer Science students seeking interactive visual tracing to learn sorting, pathfinding, and graph search paths.
  • Educators searching for framework-free, highly responsive teaching tools to illustrate algorithm states.
6. Expected Outcome :

An installable, highly responsive SPA demonstrating full-stack API integration with a 3NF normalized Oracle database.

The final product offers structured learning modules, user progress/bookmark states, and interactive visualizations for 15 core algorithms.
